\begin{description}
    \item[\texttt{main.py}:] This file serves as the central entry point of the application. Here, the main controller class \texttt{AudioToMidi} is instantiated, initialized with the file paths for the different drum components (kick, snare, hi-hat), and the automated processing pipeline is started.
    
    \item[\texttt{AudioToMidi.py}:] This file contains the \texttt{AudioToMidi} class, which acts as the central orchestrator of the workflow. It creates instances of the other processing classes and controls the sequential execution for each instrument.
    
    \item[\texttt{LoadAudioFile.py}:] The \texttt{LoadAudioFile} class encapsulates the reading of audio data using the \texttt{librosa}\cite{librosa} library. The resulting audio array and the sampling rate are stored directly within the object as its state. Additionally, the class contains custom methods for preprocessing steps, such as an implemented noise gate.
    
    \item[\texttt{LoadSTFT.py}:] This class is responsible for calculating the Short-Time Fourier Transform (STFT). The class stores the calculated spectrogram as an attribute and provides methods for direct visualization.
    
    \item[\texttt{SpectralFluxCalculator.py}:] Here, the spectral flux is calculated from the STFT data. The \texttt{SpectralFluxCalculator} class holds the STFT data as an attribute and provides methods for calculating the logarithmic magnitude as well as positive signal changes to detect high-energy transients (onsets) in the frequency spectrum.
    
    \item[\texttt{PeakPicking.py}:] The logic for the actual beat detection is isolated within the \texttt{PeakPicking} class. It internally calculates an adaptive threshold using convolution (moving average) over the spectral flux and accounts for debouncing parameters. The finally detected onset frames are stored as an object attribute.
    
    \item[\texttt{Filtering.py}:] The \texttt{Filtering} class manages various elliptical bandpass filters for the specific frequency ranges of the kick, snare, and hi-hat. It utilizes \texttt{scipy.signal}\cite{scipy} and keeps the filter states ready as instance variables for subsequent filtering processes.
    
    \item[\texttt{MidiExport.py}:] The final step of the pipeline is encapsulated in the \texttt{MidiExport} class, which handles the conversion of the detected frames into a MIDI file using the \texttt{mido}\cite{mido} library. It takes care of converting frames to seconds and then to MIDI ticks, generates individual tracks for each instrument, and bundles them on the standard MIDI drum channel.
\end{description}

\subsection{Loading Audio Data}
The Python library \texttt{librosa} is used for the actual loading of the audio data. The \texttt{load\_audio}\cite{librosa} method reads the specified file, converts it to a mono signal by default, and uses a target sample rate of 44100 Hz. 

\subsection{Noise Gate}
After the signal has been successfully loaded, noise reduction is applied to remove background noise in quiet passages. This is realized by the \texttt{noise\_gate} method, which cuts off all values below a certain threshold and sets them to 0. The noise gate threshold is calculated dynamically using percentiles.
\begin{enumerate}
    \item To obtain an envelope of the signal, the absolute values of the array are first computed: $\text{env} = |\text{self.audio\_array}|$.
    \item The threshold is then defined as the 90th percentile of this envelope using the \texttt{np.percentile(env, 90)} function\cite{numpy}. This means the threshold is set so that 90\% of the signal's amplitude values fall below it, effectively isolating the loudest 10\% of the audio (the actual drum hits).
    \item Finally, using the \texttt{np.where} function\cite{numpy}, all signal components whose absolute magnitude is smaller than this calculated threshold are filtered out and overwritten with zero. 
\end{enumerate}

\section{FFT Analyse}
    \subsection{FFT Algorithm}
        The Fast Fourier Transform (FFT) algorithm is used for computing the Discrete Fourier Transform (DFT) of a sequence. 
        The DFT of a sequence $x(n)$ of length $N$ is defined as:

        \begin{equation}
            X(k) = \sum_{n=0}^{N-1} x(n) e^{-j2\pi kn/N}
        \end{equation}
        For every frequency bin $k$, the DFT computes a complex number $X(k)$ that represents the amplitude and phase of the corresponding frequency component in the input signal.
        To calculate $N$ coefficients, $O(N^2)$ operations are required. This is because for each of the N output values, a sum of N terms must be computed. 

        \noindent The FFT algorithm separates the DFT into smaller DFTs, by recursively dividing the input sequence into even and odd indexed elements.

        \begin{equation}
            X(k) = \sum_{n=0}^{N/2-1} x(2n) e^{-j2\pi k(2n)/N} + \sum_{n=0}^{N/2-1} x(2n+1) e^{-j2\pi k(2n+1)/N}
        \end{equation}

        \noindent For better understanding, the even and odd indexed elements can be represented as, by using the substitution:

        \begin{equation}
            X_{even}(k) = \sum_{n=0}^{N/2-1} x(2n) e^{-j2\pi kn/(N/2)}
            ,
            X_{odd}(k) = \sum_{n=0}^{N/2-1} x(2n+1) e^{-j2\pi kn/(N/2)}
        \end{equation}
    
        \begin{align}
            X(k) &= X_{\mathrm{even}}(k) + e^{-j\frac{2\pi k}{N}}X_{\mathrm{odd}}(k), \\
            X\left(k+\frac{N}{2}\right)
            &= X_{\mathrm{even}}(k)
               + e^{-j\frac{2\pi}{N}\left(k+\frac{N}{2}\right)}X_{\mathrm{odd}}(k) \\
            &= X_{\mathrm{even}}(k)
               + e^{-j\frac{2\pi k}{N}}e^{-j\pi}X_{\mathrm{odd}}(k) \\
            &= X_{\mathrm{even}}(k)
               - e^{-j\frac{2\pi k}{N}}X_{\mathrm{odd}}(k),
            \qquad k=0,\dots,\frac{N}{2}-1.
            \end{align}

        \noindent By  recursively splitting the DFT into two smaller DFTs, the number of operations is reduced to $O(N \log N)$, which makes the FFT algorithm much more efficient for large input sizes.

     \subsection{Windowing}
        To generate the spectrogram, the input signal is divided into overlapping frames, and a window function is applied to each frame to minimize spectral leakage.
        Common window functions include Hanning, Hamming, and Blackman windows.
        
        \noindent The result is a 2D array where each column represents the magnitude spectrum of a frame, and each row corresponds to a specific frequency bin.

        \noindent Example of a 2D array representing frames and frequency bins:
        \[
        \mathbf{L} =
        \begin{bmatrix}
                1 & 4 & 1 & 6 \\
                3 & 2 & 2 & 1 \\
                1 & 5 & 3 & 6
        \end{bmatrix}
        \]
        \noindent This array can be visualized as a spectrogram, showing how the frequency content of the signal evolves over time.
    

\section{FLUX Detection}

    \subsection{Calculation of Logarithmic Magnitude Spectrum}

        The logarithmic magnitude spectrum is calculated using the formula:
        \begin{equation}
            L(k) = \log_{10}(|X(k)|)
        \end{equation}
        It transforms the linear magnitude spectrum into a logarithmic scale,
        which is more aligned with human auditory perception.

     \subsection{Calculation of Spectral Flux}
        The spectral flux is calculated as the difference between the logarithmic magnitude spectra of consecutive frames:
        \begin{equation}
            F(n) = \sum_{k=0}^{N-1} L_n(k) - L_{n-1}(k)
        \end{equation}
        For each frame $n$, the spectral flux $F(n)$ is computed by summing the squared differences of the logarithmic magnitude spectra $L_n(k)$ and $L_{n-1}(k)$ across all frequency bins $k$.
        The frames are typically stored in a 2D array where each column represents a frame and each row represents a frequency bin. The difference between consecutive frames can be represented as:
        \begin{equation}
            \Delta L(k) = L_n(k) - L_{n-1}(k)
        \end{equation}
        Example of a 2D array representing frames and frequency bins:
        \[
        \mathbf{L} =
        \begin{bmatrix}
                1 & 4 & 1 & 6 \\
                3 & 2 & 2 & 1 \\
                1 & 5 & 3 & 6
        \end{bmatrix}
        \]
        The resulting difference between consecutive frames is:
        \[
        \Delta \mathbf{L} =
        \begin{bmatrix}
                4 & 1 & 6 \\
                2 & 2 & 1 \\
                5 & 3 & 6
        \end{bmatrix} 
        -
        \begin{bmatrix}
                1 & 4 & 1 \\
                3 & 2 & 2 \\
                1 & 5 & 3
        \end{bmatrix} 
        =
        \begin{bmatrix}
                3 & -3 & 5 \\
                -1 & 0 & -1 \\
                4 & -2 & 3
        \end{bmatrix}
            \]
        Then, the spectral flux is computed by summing these differences across all frequency bins for each frame:
        \begin{equation}
            F(n) = \sum_{k} |\Delta L(k)|
        \end{equation}
        \[
        \mathbf{F} =
        \begin{bmatrix}
            |3| + |(-1)| + |4| &
            |(-3)| + |0| + |(-2)| &
            |5| + |(-1)| + |3|
        \end{bmatrix}^T
        =
        \begin{bmatrix}
            8 & 5 & 9
            \end{bmatrix}^T
            \]

        \begin{figure}[htpb]
            \centering
            \includegraphics[width=0.8\textwidth]{spectral_flux.png}
            \caption{Visualization of the spectral flux calculation.}
            \label{fig:spectral_flux}
        \end{figure}
        The Code to calculate the spectral flux from the differences is as follows:
        \small
\begin{lstlisting}
flux = np.sum((calculated_FLUX[:, 1:] - calculated_FLUX[:, :-1]))   
\end{lstlisting}
         \normalsize

    \subsection{Half-Wave Rectification}
        Half-wave rectification is applied to the spectral flux to retain only positive values, which correspond to increases in spectral energy. This is done by setting all negative values to zero:
        \begin{equation}
            F_{rect}(n) = \max(0, F(n))
        \end{equation}
        This step ensures that only significant increases in spectral energy are considered for further analysis.
        \\
        \\
        For example, given the spectral flux values:
        \[ \Delta L = \begin{bmatrix}
                -2 & 2 & -4 \\
                3  & 1 & 5\\
                -1 & 4 & -6
            \end{bmatrix}
            = 
            \begin{bmatrix}
                0 & 2 & 0 \\
                3 & 1 & 5\\
                0 & 4 & 0
            \end{bmatrix}
        \]
        And Summed up:
        \[ F =\begin{bmatrix}
                0+3+0 & 2+1+4 & 0+5+0
            \end{bmatrix}^T
            =
            \begin{bmatrix}
                3 & 7 & 5
            \end{bmatrix}^T
        \]

\section{STFT and Time-Frequency Analysis}

\begin{frame}{FFT and STFT}
    \begin{itemize}
        \item The \textbf{FFT} efficiently computes the Discrete Fourier Transform.
        \item Computational complexity is reduced from
        \[
        O(N^2) \rightarrow O(N \log N)
        \]
        \item For non-stationary audio, the signal is divided into overlapping frames.
        \item A window function is applied to each frame.
        \item This produces the \textbf{Short-Time Fourier Transform (STFT)}.
    \end{itemize}

    \[
    X(k)=\sum_{n=0}^{N-1}x(n)e^{-j2\pi kn/N}\qquad
    \mathbf{|X_e|} =
    \begin{bmatrix}
        1 & 4 & 1 & 6 \\
        3 & 2 & 2 & 1 \\
        1 & 5 & 3 & 6
    \end{bmatrix}
    \]

\end{frame}

\begin{frame}{STFT: Principle and Spectrogram}
    \begin{columns}[T]
        \begin{column}{0.48\textwidth}
    \begin{itemize}
        \item The signal is split into short overlapping time frames.
        \item Each frame is multiplied by a window function.
        \item Then an FFT is calculated for every frame.
    \end{itemize}
    \end{column}    
    \begin{column}{0.48\textwidth}
    \begin{itemize}
        \item The result is a time-frequency representation: the \textbf{spectrogram}.
        \item Columns represent time frames.
        \item Rows represent frequency bins.

\begin{frame}{Adaptive Threshold and Peak Picking}
    \begin{itemize}
        \item A fixed threshold is not robust enough for varying audio levels.
        \item Therefore, an \textbf{adaptive threshold} is used.
        \item It is based on a moving average:
    \end{itemize}

    \[
    y[n] = \frac{1}{N}\sum_{k=-\lfloor N/2 \rfloor}^{\lfloor (N-1)/2 \rfloor} x[n+k]
    \]

    \begin{itemize}
        \item An additional offset avoids detections from the noise floor.
        \item A valid onset must satisfy:
        \begin{enumerate}
            \item above threshold,
            \item local maximum,
            \item enough distance to the previous onset.
        \end{enumerate}
    \end{itemize}
\end{frame}

\section{Conclusion}

\begin{frame}{Conclusion and Outlook}
    \begin{itemize}
        \item The system can generate usable MIDI drum patterns from audio.
        \item It combines:
        \begin{itemize}
            \item AI-based source separation
            \item classical DSP methods
            \item rule-based onset detection
        \end{itemize}
        \item Main limitations:
        \begin{itemize}
            \item imperfect source separation
            \item overlap between drum components
            \item noise in extracted stems
        \end{itemize}
        \item Future work:
        \begin{itemize}
            \item improved separation models
            \item more robust onset detection
            \item automatic tempo estimation
        \end{itemize}
    \end{itemize}
\end{frame}
